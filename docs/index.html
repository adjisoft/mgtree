mgtree Docs

Cross-platform CLI

The tree command for people who also want search, preview, and `ls` muscle memory.

mgtree combines recursive directory views with regex filters, name and content search, live previews, colorful rendering, optional icons, and a broad set of `ls`-style flags such as `-lah`, `-aF`, `-t`, `-S`, and `-d`.

Installation

mgtree is a Go CLI project. You can run it directly from source, build a local binary, or integrate it into your own release workflow.

Run from source

go run . --help

Build a local binary

go build -o mgtree .

Windows binary name

go build -o mgtree.exe .

Quick Start

1

Print a tree

mgtree .
2

Enable long output and human sizes

mgtree -lah
3

Search by file name

mgtree -s config
4

Search inside file contents with preview

mgtree -c "token|secret" -p 2

Core Features

Regex filtering

Use include and exclude regex filters to keep the output focused on exactly the files and directories you care about.

mgtree --regex ".*\.go$"
mgtree --exclude "node_modules|.git"

Name and content search

Search file names with substring matching, or scan file contents with regular expressions.

mgtree -s auth
mgtree -c "password|token"

Preview snippets

Preview the first useful lines of a text file, or show content snippets around the first match.

mgtree --preview 3
mgtree -c handler --preview 2

Visual output

Use colors, icons, and classifier suffixes to make large trees easier to scan at a glance.

mgtree --icons --color
mgtree -F -l

`ls` Compatibility

mgtree keeps its recursive tree layout, but it also accepts familiar `ls` options so you can carry over muscle memory from Unix shells.

Flag Meaning in mgtree
-lShow long metadata columns inside the tree output.
-a, -AShow dotfiles and dot-directories that are hidden by default.
-hPrint human-readable sizes such as 2.0KB and 4.6MB.
-t, -S, -X, -USort by time, size, extension, or discovery order.
-rReverse the final sort order.
-dShow the target directory itself without descending into its children.
-FAdd `ls`-style suffixes like /, *, and @.
-R, -1Accepted for compatibility; tree output is already recursive and one item per line.

Compatibility note: because `-h`, `-r`, and `-d` now follow `ls` conventions, mgtree-specific shorthand changed to -e for --regex and -L for --depth.

Flags Reference

Flag Description

-l, -a, -h, -FEnable ls-style long output, hidden files, human sizes, and classifier suffixes.

Examples

Audit Go source files with long output

mgtree -lah -e ".*\.go$"

Search for secrets and preview nearby lines

mgtree -c "password|secret|token" -p 2

Sort by file size and reverse the order

mgtree -Sr

Show only the directory itself

mgtree -d docs

Skip heavy folders while searching by name

mgtree -x "node_modules|.git" -s config

Use icons and long mode together

mgtree -liF --icons src

Behavior Notes

  • Dotfiles are hidden by default, just like `ls`. Use `-a` or `-A` to reveal them.
  • Content search scans text files only. Binary files and large files are skipped in safe mode.
  • The output is recursive by default. `-R` is accepted for compatibility, but it does not change the result.
  • Preview lines are useful for code review, config audits, and keyword hunting without leaving the tree view.
  • When multiple sorting flags are present, the last sort flag wins, and `-r` reverses that final order.

FAQ

Is mgtree only for Go projects?

No. It is written in Go, but it works for any directory structure and any text-based source tree.

Does mgtree replace `tree` and `ls`?

It is designed to cover the common use cases of both: recursive structure from `tree`, plus convenient listing habits from `ls`.

What changed because of `ls` compatibility?

The short flags -h, -r, and -d now follow `ls` semantics. mgtree-specific depth and regex short flags use -L and -e.
